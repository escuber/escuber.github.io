Accessibility First Form Agent

Helping users complete complex web forms from photos of documents with a human in the loop. Built to reduce friction for assistive tech users and high-cognitive-load tasks.

Mission

  • Reduce friction for people using screen readers, keyboard only navigation, or motor aids.
  • Lower cognitive load for ADHD and executive function challenges.
  • Keep user control, privacy, and auditability at the core.

What it does

  • Parses photos of paper documents and page screenshots into structured data.
  • Maps values to fields across multi step flows and conditional logic.
  • Crafts guided answers to resume questions to avoid extra prompts.
  • Detects errors and performs self recovery to the right step.
  • Returns a draft for the requester to approve.

How it works

  • OpenAI vision, OCR, and LLM APIs for perception and reasoning.
  • Python agent services with page memory and retries.
  • MAUI client for capture, review, and transport.

Accessibility commitments

  • WCAG 2.2 AA targets for contrast, focus, semantics, and keyboard access.
  • Screen reader labels with aria-label, aria-describedby, and correct roles.
  • Keyboard first flows with logical tab order and visible focus rings.
  • Status messages via aria-live regions, polite for progress and assertive for errors.
  • Clear, reversible actions and explicit error prevention with field validation before submit.
  • Human in the loop: drafts always require user approval before final submit.
  • Privacy by design: minimal collection, local processing where possible, and short retention windows.

Pipeline

  1. Extract fields and options from the current page.
  2. Parse document images to structured data.
  3. Match values to fields with confidence notes and assumptions.
  4. Submit and watch for validation errors or redirects.
  5. Produce an audit log with steps, timestamps, and decisions.
